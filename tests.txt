Test methods

+ Initial Debug test

    Use a telnet client to connect to server. Type some requests in terminal and
    see if the response is expected and eyeball the server log.

+ Correctness test

    1) Use Chrome browser to view static '/'
    
    2) Use Chrome browser to run get object and add object. Check server/router log

+ Memory leak test
           
    Run server with valgrind. Initial multiple connections from brower.
    After a while kill siege and the server to see if there are **definitely lost**
    memory blocks.
                  
+ Evil test

    Send crafted request that is really large > 8192.

+ Stress test
                          
    Multiple browser and try get/add file requests.
